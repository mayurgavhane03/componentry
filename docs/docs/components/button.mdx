### Form Buttons

Use the `type` prop to control how the button interacts with a surrounding `<form>`. Set `type="submit"` to submit the form or `type="reset"` to clear it. The `name` and `value` props are submitted as part of the form data when using `type="submit"`.

<FrameworkTabs
  react={`import { CButton } from "@componentry-ui/react";

const App = () => (
  <form onSubmit={(e) => { e.preventDefault(); alert('Submitted!'); }}>
    <input name="username" placeholder="Username" />
    <CButton type="submit" variant="primary">Submit</CButton>
    <CButton type="reset" variant="neutral">Reset</CButton>
  </form>
);`}
  vue={`<script setup lang="ts">
import { CButton } from '@componentry-ui/vue';

function handleSubmit(e: Event) {
  e.preventDefault();
  alert('Submitted!');
}
</script>

<template>
  <form @submit="handleSubmit">
    <input name="username" placeholder="Username" />
    <CButton type="submit" variant="primary">Submit</CButton>
    <CButton type="reset" variant="neutral">Reset</CButton>
  </form>
</template>`}
  angular={`import { Component } from '@angular/core';
import { CButton } from '@componentry-ui/angular';

@Component({
  selector: 'app-root',
  standalone: true,
  imports: [CButton],

      <c-button type="reset" variant="neutral">Reset</c-button>
    </form>
  \`,
})
export class AppComponent {
  handleSubmit(e: Event) {
    e.preventDefault();

  }
}`}
  html={`<script type="module">

## Methods

The button exposes the following public methods for programmatic control.

| Method | Description |
|--------|-------------|
| `triggerClick()` | Simulates a click on the button. |
| `setFocus(options?)` | Sets focus on the button. |
| `removeFocus()` | Removes focus from the button. |
| `checkValidity()` | Checks validity without showing a validation message. Returns `Promise<boolean>`. |
| `reportValidity()` | Checks validity and shows the browser's validation message if invalid. Returns `Promise<boolean>`. |
| `setCustomValidity(message)` | Sets a custom validation message. Pass an empty string to restore validity. |

---

## Events

| Event | Description |
|-------|-------------|
| `cBlur` | Emitted when the button loses focus. |
| `cFocus` | Emitted when the button gains focus. |
| `cInvalid` | Emitted when the form control fails constraint validation. Cancelable. |

---

## Slots

| Slot | Description |
|------|-------------|
| _(default)_ | The button's label. |
| `prefix` | A presentational prefix icon or element placed before the label. |
| `suffix` | A presentational suffix icon or element placed after the label. |

---

## CSS Parts

Use `::part()` to style the internal shadow DOM elements.

| Part | Description |
|------|-------------|
| `base` | The component's base wrapper (`<button>` or `<a>`). |
| `prefix` | The container wrapping the prefix slot. |
| `label` | The button's label container. |
| `suffix` | The container wrapping the suffix slot. |
| `caret` | The caret icon container. |
| `spinner` | The spinner shown in loading state. |

```css
/* Example: custom primary button style */
c-button[variant="primary"]::part(base) {
  background-color: hotpink;
  border-color: hotpink;
}
```

| `loading` | `loading` | `boolean` | `false` | Draws the button in a loading state. |
| `outline` | `outline` | `boolean` | `false` | Draws an outlined button. |
| `pill` | `pill` | `boolean` | `false` | Draws a pill-style button with rounded edges. |
| `circle` | `circle` | `boolean` | `false` | Draws a circular icon button. |
| `type` | `type` | `'button' \| 'submit' \| 'reset'` | `'button'` | The native button type. |
| `name` | `name` | `string` | `''` | The name submitted with form data. |

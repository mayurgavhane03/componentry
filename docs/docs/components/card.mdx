# Card

A container for grouping related content. A card is composed of a root `c-card` element plus optional `c-card-header`, `c-card-content`, and `c-card-footer` regions.

## Anatomy

A card is assembled from the following parts. `c-card-header` exposes three **named** slots; every other part uses a default slot.

| Part | Tag | Slot |
|---|---|---|
| Root container | `c-card` | default (accepts header / content / footer) |
| Header region | `c-card-header` | named: `title`, `description`, `action` |
| Title | `c-card-title` | default — pass into header via `slot="title"` |
| Description | `c-card-description` | default — pass into header via `slot="description"` |
| Header-aligned action | `c-card-action` | default — pass into header via `slot="action"` |
| Body content | `c-card-content` | default |
| Footer region | `c-card-footer` | default |

## Props

### CCard

| Prop | Type | Default | Description |
|---|---|---|---|
| `size` | `'default' \| 'sm'` | `'default'` | Controls spacing density. `sm` reduces internal gap and padding via `--card-spacing`. |

All other parts (`c-card-header`, `c-card-title`, `c-card-description`, `c-card-action`, `c-card-content`, `c-card-footer`) take no props — they're purely structural/slot wrappers.

---

## Slots Reference

| Component | Slot | Required |
|---|---|---|
| `c-card` | default | Accepts `c-card-header`, `c-card-content`, `c-card-footer` (and any other content) in render order. |
| `c-card-header` | `title` | No — omit if the header has no title. |
| `c-card-header` | `description` | No |
| `c-card-header` | `action` | No — typically a `c-card-action` wrapping a button or link. |
| `c-card-title` | default | — |
| `c-card-description` | default | — |
| `c-card-action` | default | — |
| `c-card-content` | default | — |
| `c-card-footer` | default | — |

:::tip
`c-card-title`, `c-card-description`, and `c-card-action` only need a `slot` attribute when placed **inside** `c-card-header`. Outside of that context the attribute has no effect.
:::
